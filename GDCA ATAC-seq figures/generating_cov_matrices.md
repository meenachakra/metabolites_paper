This part of the analysis protocol was inspired by sections 3.4 and 3.9 of:
https://pubmed.ncbi.nlm.nih.gov/33606259/

# Manually edit the headers of files generated upstream

Manually edited the headers of up_peaks_input_for_cov_heatmaps.bed and down_peaks_input_for_cov_heatmaps.bed (generated by deseq2.Rmd) to match the format specified in the article (section 3.9) - #chr	left	right	baseMean	log2FoldChange

# Merged the bams for each brain per condition (created during the IDR procedure)

Used the following code. Excluded Brain 3 due to low TSS, as we did for the DESeq2 analysis.

```
samtools merge --threads 4 merged_saline_no_brain3.bam Brain_1_pooled.sorted.bam Brain_5_pooled.sorted.bam Brain_7_pooled.sorted.bam

samtools merge --threads 4 merged_gdca.bam Brain_2_pooled.sorted.bam Brain_4_pooled.sorted.bam Brain_6_pooled.sorted.bam Brain_8_pooled.sorted.bam
```

# Sorted and indexed the merged bams

```
samtools sort --threads 8 -o merged_saline_no_brain3.sorted.bam merged_saline_no_brain3.bam
samtools sort --threads 8 -o merged_gdca.sorted.bam merged_gdca.bam
samtools index --threads 8 merged_saline_no_brain3.sorted.bam
samtools index --threads 8 merged_gdca.sorted.bam
```

# Generated bigwig files

Used deeptools v3.3.0 to execute the following:

```
bamCoverage --bam merged_saline_no_brain3.sorted.bam -o merged_saline_no_brain3.sorted.bigWig --binSize 100 --normalizeUsing CPM --numberOfProcessors 8

bamCoverage --bam merged_gdca.sorted.bam -o merged_gdca.sorted.bigWig --binSize 100 --normalizeUsing CPM --numberOfProcessors 8
```

# Generated coverage matrices for peaks with increased accessibility in GDCA-treated brains

First, downloaded the following script: https://raw.githubusercontent.com/georgimarinov/GeorgiScripts/refs/heads/master/signalAroundCoordinate-individual.py

Used the python program 2to3 to convert it to python 3. The conversion was executed using Python 3.10.

Also commented out the line "from sets import Set"

Then created the following environment.

```
mamba create -n peaks -c conda-forge -c bioconda python=3.11 numpy pybigwig
```

Within the peaks environment, executed the following commands.

```
python signalAroundCoordinate-individual.py up_peaks_input_for_cov_heatmaps.bed 0 midPoint -noStrand 2000 2000 merged_saline_no_brain3.sorted.bigWig up_peaks_cov_in_saline_samples.matrix -sortby 4

python signalAroundCoordinate-individual.py up_peaks_input_for_cov_heatmaps.bed 0 midPoint -noStrand 2000 2000 merged_gdca.sorted.bigWig up_peaks_cov_in_gdca_samples.matrix -sortby 4
```

# Generated coverage matrices for peaks with decreased accessibility in GDCA-treated brains

Within the peaks environment created above, executed the following commands.

```
python signalAroundCoordinate-individual.py down_peaks_input_for_cov_heatmaps.bed 0 midPoint -noStrand 2000 2000 merged_saline_no_brain3.sorted.bigWig down_peaks_cov_in_saline_samples.matrix -sortby 4

python signalAroundCoordinate-individual.py down_peaks_input_for_cov_heatmaps.bed 0 midPoint -noStrand 2000 2000 merged_gdca.sorted.bigWig down_peaks_cov_in_gdca_samples.matrix -sortby 4
```
